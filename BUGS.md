
# Баг-репорты

В этом файле описаны проблемы, которые я нашёл при тестировании API.

---

## БАГ-001: API не даёт создать объявление

### Краткое описание
При попытке создать объявление сервер всегда возвращает ошибку "поле likes обязательно", даже когда мы передаём это поле.

### Серьёзность
**КРИТИЧЕСКАЯ** — основная функция (создание объявлений) не работает

### Шаги воспроизведения
1. Отправить запрос на создание объявления:
```bash
curl -X POST https://qa-internship.avito.com/api/1/item \
  -H "Content-Type: application/json" \
  -d '{"sellerId": 123456, "name": "Телефон", "price": 10000, "likes": 0}'
```

### Фактический результат
Сервер возвращает ошибку 400 с сообщением:
```json
{"result":{"message":"поле likes обязательно","messages":{}},"status":"400"}
```

### Ожидаемый результат
Сервер возвращает 200 OK и ID созданного объявления:
```json
{"id": "123456789", "status": "ok"}
```

### Какие тесты затронуты
Из-за этого бага упали 8 тестов:
- P-01: Создание объявления с правильными данными
- P-02: Получение объявления по ID
- P-04: Получение статистики
- P-07: Создание объявления с русскими буквами
- C-01: Повторное создание того же объявления
- C-02: Повторный запрос одного объявления
- C-04: Лишние поля в запросе
- E2E-01: Полный жизненный цикл объявления

### Окружение
- URL: https://qa-internship.avito.com
- Метод: POST /api/1/item
- Дата обнаружения: 2026-04-03

### Дополнительная информация
Я пробовал передавать поле `likes` в разных форматах:
- `"likes": 0` на верхнем уровне
- `"statistics": {"likes": 0}` внутри объекта
- `"likes": "0"` как строку

Ни один вариант не сработал. API всегда отвечает одинаковой ошибкой.

## БАГ-002: Неправильный ответ при запросе всех объявлений продавца

### Краткое описание
При запросе всех объявлений продавца в ответе нет поля `sellerId`, из-за чего тест не может проверить, что объявления принадлежат правильному продавцу.

### Серьёзность
**СРЕДНЯЯ** — данные можно получить, но нельзя проверить их принадлежность

### Шаги воспроизведения
1. Создать объявление для продавца с ID 123456
2. Запросить все объявления этого продавца:
```bash
curl -X GET https://qa-internship.avito.com/api/1/123456/item
```

### Фактический результат
Сервер возвращает список объявлений, но внутри каждого объявления нет поля `sellerId`:
```json
[
  {
    "id": "123",
    "name": "Телефон",
    "price": 10000,
    "likes": 0
  }
]
```

### Ожидаемый результат
В каждом объявлении должно быть поле `sellerId`, чтобы знать, кто продавец:
```json
[
  {
    "id": "123",
    "sellerId": 123456,
    "name": "Телефон",
    "price": 10000,
    "likes": 0
  }
]
```

### Какие тесты затронуты
- P-03: Получение всех объявлений продавца

### Окружение
- URL: https://qa-internship.avito.com
- Метод: GET /api/1/{sellerId}/item
- Дата обнаружения: 2026-04-03

## БАГ-003: API не принимает цену 0

### Краткое описание
При попытке создать бесплатное объявление с ценой 0 сервер возвращает ошибку "поле price обязательно", хотя поле price передаётся.

### Серьёзность
**НИЗКАЯ** — бесплатные объявления не являются основной функцией

### Шаги воспроизведения
1. Отправить запрос на создание объявления с ценой 0:
```bash
curl -X POST https://qa-internship.avito.com/api/1/item \
  -H "Content-Type: application/json" \
  -d '{"sellerId": 123456, "name": "Бесплатно", "price": 0, "likes": 0}'
```

### Фактический результат
Сервер возвращает ошибку:
```json
{"result":{"message":"поле price обязательно","messages":{}},"status":"400"}
```

### Ожидаемый результат
Объявление с ценой 0 создаётся (бесплатные объявления должны быть разрешены)

### Какие тесты затронуты
- P-05: Создание объявления с ценой 0

### Окружение
- URL: https://qa-internship.avito.com
- Метод: POST /api/1/item
- Дата обнаружения: 2026-04-03
