
# Тест-кейсы для API объявлений Avito

## О чем этот документ

Здесь описаны все тесты, которые проверяют работу API объявлений.

**Хост для тестов:** https://qa-internship.avito.com

---

## Как устроены тесты

Все тесты лежат в папке `api_tests/`. Вот какие файлы за что отвечают:

| Файл | Что проверяет |
|------|---------------|
| test_positive.py | Правильные запросы (должны работать) |
| test_negative.py | Неправильные запросы (должны выдавать ошибку) |
| test_idempotency.py | Повторные и параллельные запросы |
| test_e2e.py | Полную цепочку действий |
| test_performance.py | Скорость работы и нагрузку |

**Всего тестов:** 21

---

## ПОЗИТИВНЫЕ ТЕСТЫ (7 тестов)

Эти тесты проверяют, что API правильно работает с правильными данными.

### Файл: `test_positive.py`

#### Тест P-01: Создание объявления с правильными данными
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveCreate` → метод `test_create_ad_success`

**Что делаем:** Отправляем запрос на создание объявления со всеми нужными полями:
- sellerId (номер продавца)
- name (название)
- price (цена)
- likes, viewCount, contacts (статистика)

**Что ожидаем:** Сервер возвращает ID нового объявления

**Результат:** ❌ ПРОВАЛЕН (баг в API)

---

#### Тест P-02: Получение объявления по его ID
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveGet` → метод `test_get_ad_by_id_success`

**Что делаем:** Сначала создаём объявление, потом просим показать его по ID

**Что ожидаем:** Получаем то же самое объявление, которое создали

**Результат:** ❌ ПРОВАЛЕН (баг в API)

---

#### Тест P-03: Получение всех объявлений продавца
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveGet` → метод `test_get_ads_by_seller_success`

**Что делаем:** Создаём 2 объявления для одного продавца, потом запрашиваем все его объявления

**Что ожидаем:** Получаем список, в котором есть оба созданных объявления

**Результат:** ❌ ПРОВАЛЕН (ошибка KeyError: 'sellerId')

---

#### Тест P-04: Получение статистики объявления
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveGet` → метод `test_get_statistics_success`

**Что делаем:** Создаём объявление с определённой статистикой (лайки, просмотры), потом запрашиваем статистику

**Что ожидаем:** Получаем ту же статистику, которую указали

**Результат:** ❌ ПРОВАЛЕН (баг в API)

---

#### Тест P-05: Создание объявления с ценой 0
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveCreate` → метод `test_create_ad_zero_price`

**Что делаем:** Создаём бесплатное объявление (цена = 0)

**Что ожидаем:** Объявление создаётся, цена сохраняется как 0

**Результат:** ⏭️ ПРОПУЩЕН (API не поддерживает цену 0)

---

#### Тест P-06: Создание объявления с максимальной ценой
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveCreate` → метод `test_create_ad_max_price`

**Что делаем:** Создаём очень дорогое объявление (999999999)

**Что ожидаем:** Объявление создаётся, цена сохраняется

**Результат:** ⏭️ ПРОПУЩЕН (баг в API)

---

#### Тест P-07: Создание объявления с русскими буквами
**Где лежит:** `api_tests/test_positive.py` → класс `TestPositiveCreate` → метод `test_create_ad_russian_name`

**Что делаем:** Создаём объявление с названием на русском языке

## E2E ТЕСТ (1 тест)

Сквозной тест, который проверяет всю цепочку действий.

### Файл: `test_e2e.py`

#### Тест E2E-01: Полный жизненный цикл объявления
**Где лежит:** `api_tests/test_e2e.py` → класс `TestE2E` → метод `test_complete_lifecycle`

**Что делаем:** Проходим весь путь:
1. Создаём объявление
2. Получаем его по ID
3. Получаем статистику
4. Получаем все объявления продавца

**Что ожидаем:** Все шаги успешны

**Результат:** ❌ ПРОВАЛЕН (баг в API на первом шаге)
