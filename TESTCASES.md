# Тест-кейсы для API объявлений

## Обзор
- **Базовый URL**: `https://qa-internship.avito.com`
- **API версии**: v1 (`/api/1/*`) и v2 (`/api/2/*`)
- **Основные эндпоинты**:
  - POST `/api/1/item` - создание объявления
  - GET `/api/1/item/{id}` - получение объявления по ID
  - GET `/api/1/{sellerId}/item` - получение объявлений продавца
  - GET `/api/1/statistic/{id}` - получение статистики

## Техники тест-дизайна
- **Классы эквивалентности**: валидные/невалидные значения полей
- **Граничные значения**: min/max price, sellerId, длина name
- **Попарное тестирование**: комбинации параметров
- **Предугадывание ошибок**: спецсимволы, SQL injection, XSS

## Позитивные тест-кейсы (P)

### P-01: Создание объявления с валидными данными
| Поле | Значение |
|------|----------|
| name | "iPhone 15 Pro" |
| price | 99999 |
| sellerId | 123456 |
| statistics | {"likes": 0, "viewCount": 0, "contacts": 0} |

**Ожидаемый результат**: 
- Status 200 OK
- Возвращается объект с полями: id, sellerId, name, price, statistics, createdAt
- id не пустой и уникальный

### P-02: Получение существующего объявления по ID
**Предусловие**: Создано объявление с известными данными
**Ожидаемый результат**:
- Status 200 OK
- Данные совпадают с созданными
- Присутствуют все поля

### P-03: Получение всех объявлений продавца
**Предусловие**: Создано 3 объявления для sellerId=123456
**Ожидаемый результат**:
- Status 200 OK
- Возвращается массив из 3 объявлений
- У каждого объявления sellerId=123456

### P-04: Получение статистики объявления
**Предусловие**: Создано объявление
**Ожидаемый результат**:
- Status 200 OK
- Возвращается массив со статистикой (likes, viewCount, contacts)
- Статистика не отрицательная

### P-05: Создание объявления с минимальной ценой (0)
**Ожидаемый результат**: Status 200, price=0

### P-06: Создание объявления с максимальной ценой (999999999)
**Ожидаемый результат**: Status 200, цена сохранена корректно

### P-07: Создание объявления с максимальной длиной имени (255 символов)
**Ожидаемый результат**: Status 200, имя сохраняется полностью

### P-08: Создание объявления с русскими символами в имени
**Ожидаемый результат**: Status 200, имя отображается корректно

### P-09: Создание объявления с эмодзи в имени
**Ожидаемый результат**: Status 200, эмодзи сохраняются

### P-10: Создание объявления со спецсимволами (!@#$%^&*())
**Ожидаемый результат**: Status 200, спецсимволы сохраняются

**Ожидаемый результат**: Все создаются успешно, ID уникальны

## Функциональные проверки (F)

### F-01: E2E сценарий - полный жизненный цикл объявления
1. Создать объявление → получить ID
2. Получить объявление по ID → проверить данные
3. Получить статистику → проверить начальные значения
4. Получить все объявления продавца → найти созданное
5. (Опционально) Удалить объявление → проверить 404

### F-02: Проверка уникальности ID
**Ожидаемый результат**: При создании 100 объявлений все ID разные

### F-03: Проверка формата createdAt
**Ожидаемый результат**: Дата в формате ISO 8601 (YYYY-MM-DDTHH:MM:SSZ)

### F-04: Проверка типа полей в ответе
- id: string
- sellerId: integer
- name: string
- price: integer
- statistics: object

**Ожидаемый результат**: GET /{sellerId}/item возвращает результат за разумное время

**Ожидаемый результат**: При ошибке 500 возвращается понятный JSON, а не stack trace
