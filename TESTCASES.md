# Тест-кейсы для API объявлений Avito

## О чем этот документ

Здесь описаны все тесты, которые проверяют работу API объявлений. Каждый тест проверяет конкретную ситуацию: что должно работать, а что нет.

**Хост для тестов:** https://qa-internship.avito.com

**Что такое API:** Это способ, которым программа общается с другой программой. Мы отправляем запросы и смотрим на ответы.

---

## Как устроены тесты

Все тесты написаны на Python в папке `api_tests/`. Вот какие файлы за что отвечают:

| Файл | Что проверяет |
|------|---------------|
| test_positive.py | Правильные запросы (должны работать) |
| test_negative.py | Неправильные запросы (должны выдавать ошибку) |
| test_idempotency.py | Повторные запросы и особые случаи |
| test_e2e.py | Полную цепочку действий (создать -> найти -> посмотреть статистику) |
| test_performance.py | Скорость работы и нагрузку |

---

## ПОЗИТИВНЫЕ ТЕСТЫ (что должно работать)

Эти тесты проверяют, что API правильно обрабатывает правильные данные.

### test_positive.py

#### Тест P-01: Создание объявления с правильными данными
**Файл:** `api_tests/test_positive.py` → `TestPositiveCreate.test_create_ad_success`

**Что делаем:** Отправляем запрос на создание объявления со всеми нужными полями:
- sellerId (номер продавца)
- name (название)
- price (цена)
- likes, viewCount, contacts (статистика)

**Что ожидаем:** Сервер возвращает ID нового объявления и статус 200 (всё хорошо)

**Почему важно:** Это основная функция - создавать объявления

## ТЕКУЩИЙ СТАТУС ТЕСТОВ

Запустив тесты, вы увидите:

- **9 тестов прошло** (все негативные - проверка ошибок работает)
- **9 тестов упало** (позитивные - потому что API не работает)
- **3 теста пропущено** (особые случаи)

**Почему падают позитивные тесты?** Потому что в API есть баг - он всегда говорит "поле likes обязательно", даже когда мы его передаём. Это описано в файле BUGS.md.

**Что это значит?** Тесты работают правильно и нашли реальную проблему в API.
