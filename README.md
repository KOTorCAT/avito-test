# API тесты для Avito

Автотесты для проверки API объявлений Avito (https://qa-internship.avito.com)

## Что проверяют тесты

- Создание объявления
- Получение объявления по ID
- Получение всех объявлений продавца
- Получение статистики

## Результаты тестирования

| Показатель | Значение |
|------------|----------|
| Всего тестов | 21 |
| ✅ Пройдено | 9 |
| ❌ Провалено | 9 |
| ⏭️ Пропущено | 3 |

### Что работает (9 тестов)
- Все негативные тесты (проверка ошибок)
- Параллельное создание объявлений
- Создание 10 объявлений подряд

### Что не работает (9 тестов)
- Создание объявления с правильными данными
- Получение объявления по ID
- Получение статистики
- Повторные запросы
- Полный сценарий (E2E)

**Причина:** API всегда возвращает ошибку "поле likes обязательно" (баг #001)

## Быстрый старт

### 1. Установка Python
```bash
# Проверьте, что Python 3.9+ установлен
python3 --version
```

### 2. Скачать проект
```bash
git clone <ваш репозиторий>
cd avito-test
```

### 3. Создать виртуальное окружение
```bash
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows
```

### 4. Установить зависимости
```bash
pip install -r requirements.txt
```

### 5. Запустить тесты
```bash
pytest -v
```

## Команды для запуска

```bash
# Все тесты
pytest -v

# Только позитивные (должны работать)
pytest api_tests/test_positive.py -v

# Только негативные (проверка ошибок)
pytest api_tests/test_negative.py -v

# Только быстрая проверка
pytest -m smoke

```

## Структура проекта

```
avito-test/
├── api_tests/           # Все тесты
│   ├── test_positive.py    # Позитивные (7 тестов)
│   ├── test_negative.py    # Негативные (7 тестов)
│   ├── test_idempotency.py # Повторные запросы (4 теста)
│   ├── test_e2e.py         # Полный сценарий (1 тест)
│   ├── test_performance.py # Скорость и нагрузка (2 теста)
│   ├── conftest.py         # Настройки для тестов
│   └── utils/              # Вспомогательный код
├── TESTCASES.md        # Описание всех тестов
├── BUGS.md             # Найденные проблемы
├── requirements.txt    # Нужные библиотеки
└── pytest.ini          # Настройки pytest
```

## Что в каждом файле

| Файл | Что делает |
|------|------------|
| test_positive.py | Проверяет, что API работает с правильными данными |
| test_negative.py | Проверяет, что API ругается на неправильные данные |
| test_idempotency.py | Проверяет повторные и параллельные запросы |
| test_e2e.py | Проверяет полный сценарий (создать → найти → статистика) |
| test_performance.py | Проверяет скорость и нагрузку |

## Известные проблемы

Из-за бага в API (BUGS.md) следующие тесты падают:
- Создание объявления
- Получение объявления по ID
- Получение статистики
- Повторные запросы
- Полный сценарий

Это не ошибка в тестах, а реальная проблема в API.
